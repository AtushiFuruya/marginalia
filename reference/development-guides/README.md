# Development Guides - 開発ガイド

このディレクトリには、開発プロセス、ツール使用方法、ベストプラクティスに関するガイドが含まれています。

---

## 📁 含まれるガイド

### 1. prompting-manual.md - AIプロンプティングマニュアル
**AIを最大限活用するための必読書**

**内容:**
- 開発フロー全体像
- Issue分割の方法
- アセット準備の指示方法
- Git/package.json/.gitignoreの説明
- プロンプティングのベストプラクティス（5原則）
  1. 具体的であること
  2. コンテキストを含めること
  3. 参照を明示すること
  4. 検証可能であること
  5. 段階的であること
- Issue分割のテンプレート
- デバッグ・トラブルシューティングのプロンプト
- 統合ワークフロー
- メモリシステムの活用

**いつ使う:**
- Claude CodeやCodex CLIに指示を出す前
- 曖昧な指示で失敗した時
- Issueを作成する時
- 効率的な開発方法を知りたい時

**例:**
```
❌ 悪い指示: "サイトを作ってください"
✅ 良い指示: "年齢確認ページ（index.html）を作成してください。
             要件: reference/requirements.md を参照"
```

---

### 2. conversation-analysis.md - 会話履歴の分析
**これまでの開発プロセスの振り返り**

**内容:**
- OpenMemoryの利用可否
- 会話フローのタイムライン
- 開発視点での評価
  - ✅ 優れている点
  - ⚠️ 不合理な点・欠落要素
- 優先順位付きタスクリスト
- 推奨される次のアクション
- 総合評価

**いつ使う:**
- 開発プロセスを振り返る時
- 次のステップを決める時
- 問題点を把握したい時
- プロジェクトの進捗確認時

**重要な指摘:**
- 計画フェーズは完璧だが、実装着手が必要
- アセットファイルの準備が次の課題
- Issue分割後に実装を開始する流れが正しい

---

### 3. openmemory-setup-guide.md - OpenMemoryセットアップ
**AIの記憶機能を有効にする**

**内容:**
- OpenMemoryとは
- インストール手順
- 設定ファイルの編集方法
- 動作確認方法
- トラブルシューティング

**いつ使う:**
- OpenMemoryを初めて使う時
- 設定がうまくいかない時
- 他のAIツールと統合したい時

**設定ファイル:**
```json
{
  "mcpServers": {
    "openmemory": {
      "command": "npx",
      "args": ["-y", "@openmemory/server"],
      "env": {
        "OPENMEMORY_API_KEY": "your-key"
      }
    }
  }
}
```

---

### 4. shared-memory-architecture.md - 共有メモリアーキテクチャ
**Claude CodeとCodex CLIでメモリを共有する仕組み**

**内容:**
- アーキテクチャ概要
- ローカルメモリ + OpenMemoryの二重構造
- ディレクトリ構造（`.memory/`）
- メモリフォーマット例
- 動作フロー（保存/取得）
- メモリ管理コマンド
- セキュリティとプライバシー

**いつ使う:**
- 複数のAIツールを併用する時
- プロジェクト情報を永続化したい時
- コンテキストを自動的に共有したい時

**仕組み:**
```
ユーザー指示
    ↓
ローカルメモリに保存（.memory/）
    ↓
OpenMemoryに同期（クラウド）
    ↓
Claude Code & Codex CLIの両方からアクセス可能
```

## 🎯 ガイド利用シーン別

### シーン1: AIに効率的に指示を出したい
→ `prompting-manual.md` を読む
→ 「プロンプティングのベストプラクティス」を参照
→ テンプレートを使ってIssue作成

### シーン2: プロジェクトの進捗を確認したい
→ `conversation-analysis.md` を読む
→ 「優先順位付きタスクリスト」を確認
→ 次のアクションを決定

### シーン3: OpenMemoryを使いたい
→ `openmemory-setup-guide.md` でセットアップ
→ `shared-memory-architecture.md` で使い方を理解
→ プロジェクト情報を記憶させる

### シーン4: 開発プロセスを改善したい
→ `prompting-manual.md` の「統合ワークフロー」を確認
→ `conversation-analysis.md` で問題点を把握
→ プロセスを調整

## 📚 推奨学習パス

### 初心者向け
1. `prompting-manual.md` の「Section 1-2」を読む
2. `conversation-analysis.md` でプロジェクトの現状を把握
3. 簡単な指示から始める

### 中級者向け
1. `prompting-manual.md` の全セクションを読む
2. `openmemory-setup-guide.md` でOpenMemoryをセットアップ
3. Issue分割テンプレートを活用

### 上級者向け
1. `shared-memory-architecture.md` を読んで統合システムを構築
2. カスタムスクリプトを作成（`.memory-scripts/`）
3. プロセスを自動化

## 💡 ベストプラクティス

### DO（やるべきこと）
- ✅ 指示は具体的に
- ✅ 参照ファイルを明示
- ✅ 検証可能な成果物を定義
- ✅ 段階的に進める
- ✅ OpenMemoryで重要情報を記憶

### DON'T（やらないこと）
- ❌ 曖昧な指示
- ❌ 一度に多くを要求
- ❌ コンテキストなしの質問
- ❌ 検証方法を示さない
- ❌ エラーメッセージを省略
