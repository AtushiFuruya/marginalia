## 🔄 アニメーションの仕組み

### HTML/CSS/JavaScript の統合

**3層構造で実現**:

#### 1. HTML - 骨組み

```
セクション
  └ スライダーコンテナ
      └ トラック（移動する本体）
          └ 個別スライド（6枚）
```

各スライドには画像、名前、CV表記が含まれる。

---

#### 2. CSS - 見た目とアニメーション

**初期状態**:
- 全スライド: 透明（`opacity: 0`）、右に50px移動（`transform: translateX(50px)`）

**アクティブ状態**（`.is-active` クラス付与時）:
- 不透明（`opacity: 1`）、元の位置（`transform: translateX(0)`）
- 0.8秒かけてフェードイン＋スライドイン

**スライド移動**:
- トラック全体が `transform: translateX(-n%)` で横移動
- **1.2〜1.8秒でゆっくり移動**（画面サイズで速度変化）
- イージング: `cubic-bezier(0.25, 0.46, 0.45, 0.94)`（滑らかな加速・減速）

**ホバーエフェクト**:
- リンク全体: 5%拡大（`scale(1.05)`）
- 画像: 10%拡大（`scale(1.1)`）
- 0.3〜0.6秒で素早く反応

---

#### 3. JavaScript - 動きの制御

**主な機能**:

1. **スライド切り替え**
   - トラックを `transform: translateX()` で移動
   - 現在のスライドに `.is-active` クラスを付与
   - 前のスライドからクラスを削除

2. **自動再生**
   - 5秒ごとに次のスライドへ自動遷移
   - `setInterval` で実装

3. **ホバー時の挙動**
   - マウスがスライダー上にある間は自動再生を停止
   - マウスが離れると自動再生を再開

4. **ナビゲーション**
   - サムネイルクリックで該当スライドへジャンプ
   - クリック後、自動再生をリセット（最初から5秒カウント）

5. **レスポンシブ対応**
   - 画面幅を判定（モバイル/タブレット/デスクトップ）
   - 表示枚数に応じて移動距離を計算（1枚なら100%、2枚なら50%、3枚なら33.333%）

---

## 📱 レスポンシブ戦略

| 画面幅 | 表示枚数 | 移動速度 | 説明 |
|--------|---------|---------|------|
| **モバイル** (0-767px) | 1枚 | 1.2秒 | 速め |
| **タブレット** (768-1023px) | 2枚 | 1.5秒 | 中間 |
| **デスクトップ** (1024px+) | 3枚 | 1.8秒 | ゆっくり |

**理由**: 画面が大きいほど、ゆっくり移動させることで「ゆったり感」を演出。

### イージング関数の役割

`cubic-bezier(0.25, 0.46, 0.45, 0.94)`:
- 開始: ゆっくり加速
- 中間: 一定速度
- 終了: ゆっくり減速

**結果**: 急な動きがなく、「ゆっくり移動」している印象を与える。

## 📊 技術的な統合の仕組み

### フロー全体

```
1. ページ読み込み
   ↓
2. JavaScript初期化
   - 最初のスライドに .is-active 付与
   - 自動再生開始（5秒タイマー）
   ↓
3. アニメーション発火
   - CSS transition が .is-active 検知
   - opacity: 0→1, transform: translateX(50px)→0
   - 0.8秒かけて実行
   ↓
4. 5秒経過
   - JavaScript が次のスライドへ切り替え
   - トラック全体を translateX で移動（1.5秒）
   - .is-active を次のスライドへ移動
   ↓
5. ユーザーアクション（オプション）
   - ホバー → 自動再生停止
   - サムネイルクリック → 該当スライドへジャンプ
   - マウス離脱 → 自動再生再開
   ↓
6. ループ（3に戻る）
```

---

### CSS Transitionの役割

**宣言部分**（CSS）:
```css
.member__slider__track {
  transition: transform 1.5s cubic-bezier(...);
}
```

**トリガー部分**（JavaScript）:
```javascript
track.style.transform = `translateX(-100%)`;
// ↑この変更を検知してCSSが1.5秒かけてアニメーション
```

**ポイント**: JavaScriptは「値を変更するだけ」、実際のアニメーションはCSSが担当。

---

### JavaScriptとCSSの役割分担

| 担当 | JavaScript | CSS |
|------|-----------|-----|
| **構造** | - | HTML階層 |
| **見た目** | - | 色、サイズ、配置 |
| **アニメーション** | - | transition, transform |
| **タイミング制御** | ✅ | - |
| **状態管理** | ✅ (.is-active付与) | - |
| **ユーザー操作** | ✅ (click, hover検知) | - |
| **レスポンシブ判定** | ✅ (window.innerWidth) | メディアクエリ |

**効率的な分業**: JavaScriptは「いつ、どのスライドを」決定、CSSは「どのように動かすか」実行。
